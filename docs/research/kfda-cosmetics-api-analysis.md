# 식약처(MFDS) 화장품 공공 API 분석

> 작성일: 2026-03-09
> 목적: COSFIT 플랫폼에서 활용 가능한 식약처 화장품 관련 공공 API 조사 및 분석

---

## 1. 사용 가능한 API 목록

### 1.1 I0030 - 화장품 전성분 표시정보 (현재 COSFIT에서 사용 중)

| 항목 | 내용 |
|------|------|
| **서비스명** | 화장품 전성분 표시정보 |
| **서비스ID** | I0030 |
| **제공기관** | 식품의약품안전처 |
| **포털** | 식품안전나라 OpenAPI (openapi.foodsafetykorea.go.kr) |
| **데이터 형식** | JSON, XML |
| **총 데이터 건수** | 약 15만~20만건 (지속 증가) |

**제공 데이터 필드:**

| 필드명 | 설명 | COSFIT 활용도 |
|--------|------|--------------|
| `PRDLST_REPORT_NO` | 품목보고번호 (고유 식별자) | ProductMaster.kfdaReportNo |
| `PRDLST_NM` | 제품명 | ProductMaster.name |
| `BSSH_NM` | 업소명 (제조/수입사) | Brand.name |
| `PRDLST_DCNM` | 제품 구분명 (카테고리) | ProductMaster.category 매핑 |
| `RAWMTRL_NM` | **전성분 목록** (콤마 구분) | rawIngredients + 성분 파싱 |
| `STDR_STND` | 기준 규격 | 참고용 |
| `PRMS_DT` | 허가/보고일자 | kfdaData.permitDate |
| `UPDATE_DT` | 수정일자 | 증분 동기화 기준 |
| `EE_END_DE` | 유효기한 | 참고용 |
| `CNTC_NO` | 연락처 | 미사용 |

**핵심 장점:** 전성분(RAWMTRL_NM) 정보를 제공하는 유일한 공식 API

**엔드포인트 패턴:**
```
https://openapi.foodsafetykorea.go.kr/api/{API_KEY}/I0030/json/{시작행}/{끝행}
```

### 1.2 C003 - 화장품 제조업/수입업 현황

| 항목 | 내용 |
|------|------|
| **서비스ID** | C003 |
| **포털** | 식품안전나라 OpenAPI |
| **용도** | 화장품 제조업/수입업 등록 현황 |

**제공 데이터 필드:**
- 업소명, 대표자명, 소재지, 허가번호, 업종구분(제조/수입)
- 허가일자, 품목류

**COSFIT 활용:** Brand 정보 보강 (제조사/수입사 구분, 소재지)

### 1.3 I0040 - 기능성화장품 심사 정보

| 항목 | 내용 |
|------|------|
| **서비스ID** | I0040 |
| **포털** | 식품안전나라 OpenAPI |
| **용도** | 기능성화장품 심사/보고 결과 |

**제공 데이터 필드:**
- 제품명, 업소명, 기능성 유형 (미백/주름/자외선차단 등)
- 주요 성분, 함량, 심사결과, 심사일자

**COSFIT 활용:** 기능성 화장품 여부 판단, 기능성 성분 함량 데이터 확보

### 1.4 의약외품 허가/신고 정보 (화장품 인접)

| 항목 | 내용 |
|------|------|
| **서비스ID** | I0010 등 |
| **용도** | 의약외품(선크림, 핸드워시 등) 허가 정보 |

**COSFIT 활용:** 선크림 등 의약외품 분류 제품의 성분 정보 보강

### 1.5 화장품 원료 기준 (화장품 성분 사전)

| 항목 | 내용 |
|------|------|
| **포털** | data.go.kr (공공데이터포털) |
| **용도** | 화장품에 사용 가능/금지/제한 원료 목록 |

**제공 데이터:**
- 배합 금지 원료 목록
- 배합 한도 원료 (제한 성분 + 최대 허용 농도)
- 보존제 허용 목록
- 자외선 차단 성분 목록
- 색소 허용 목록

**COSFIT 활용:** Ingredient 안전등급(safetyGrade) 자동 산정, 규제 준수 확인

## 2. 전성분 정보 포함 여부 요약

| API | 전성분 제공 | 성분 함량 | 비고 |
|-----|:---------:|:--------:|------|
| I0030 (전성분 표시정보) | O | X | **콤마 구분 전성분 목록** |
| C003 (제조/수입업) | X | X | 업소 정보만 |
| I0040 (기능성화장품) | 부분 | 부분 | 기능성 주성분+함량만 |
| 원료 기준 | - | 한도만 | 원료 규제 목록 |

> **결론:** I0030이 전성분 목록을 제공하는 유일한 API.
> 다만 **성분 함량(농도)** 정보는 어떤 API에서도 제공하지 않음.
> 성분 순서(orderIndex)로 대략적 함량 추정 가능 (전성분 표시 규정: 함량 높은 순 기재).

## 3. API 키 발급 방법

### 식품안전나라 OpenAPI (I0030, C003, I0040)

1. **식품안전나라** (https://various.foodsafetykorea.go.kr) 회원가입
2. **OpenAPI > 인증키 신청** 메뉴에서 서비스 선택
3. 신청 후 **즉시 발급** (별도 승인 불필요)
4. 환경변수: `KFDA_API_KEY`에 설정

### 공공데이터포털 (data.go.kr) API

1. **공공데이터포털** (https://www.data.go.kr) 회원가입
2. 해당 API 상세 페이지에서 **활용신청**
3. 자동 승인 또는 1~2일 내 승인
4. 마이페이지에서 인증키 확인

## 4. 일일 호출 제한

| 포털 | 일일 한도 | 건당 최대 행 | 비고 |
|------|----------|-------------|------|
| 식품안전나라 (I0030 등) | **10,000회/일** | 1,000행 | 초당 5회 제한 |
| 공공데이터포털 (data.go.kr) | **1,000회/일** (기본) | API마다 상이 | 트래픽 신청 시 증량 가능 |

**현재 COSFIT 설정:**
- 페이지 크기: 1,000행
- Rate Limit: 250ms 간격 (초당 4회, 마진 포함)
- Full Sync 시 약 150~200 페이지 필요 → 일일 한도 내 충분

## 5. COSFIT 활용 방안

### 5.1 현재 구현 상태 (I0030)
- `src/lib/kfda/client.ts` — API 클라이언트 (페이지네이션, 재시도, Rate Limit)
- `src/lib/kfda/full-sync.ts` — 전체/일일 증분 동기화
- `src/lib/kfda/sync-job.ts` — 동기화 작업 관리
- ProductMaster에 `kfdaReportNo`, `rawIngredients`, `kfdaData` 저장

### 5.2 추가 활용 제안

#### (1) I0040 기능성화장품 연동
```
우선순위: 높음
효과: 기능성 화장품 주성분 함량 데이터 확보
구현: KfdaClient에 I0040 서비스 추가, ProductMaster.kfdaData에 병합
```

#### (2) 원료 기준 DB 구축
```
우선순위: 높음
효과: 금지/제한 성분 자동 판별 → safetyGrade 자동 산정
구현: 배합 금지/제한 원료 데이터를 Ingredient 모델에 매핑
```

#### (3) C003 제조/수입업체 정보 연동
```
우선순위: 중간
효과: Brand 정보 보강 (국내 제조 vs 수입, 제조사 소재지)
구현: Brand 모델에 제조사/수입사 정보 추가
```

#### (4) 성분 정규화 파이프라인 개선
```
현재: rawIngredients → 콤마 분리 → Ingredient DB 매칭
개선안:
  1. INCI명-한국명 매핑 테이블 구축 (식약처 원료 기준 활용)
  2. IngredientSynonym에 식약처 공식 한국명 추가
  3. 매칭률 모니터링 (현재 dataStatus QUALITY_ISSUE 활용)
```

### 5.3 데이터 갱신 전략

| 동기화 유형 | 주기 | 대상 |
|------------|------|------|
| Full Sync | 월 1회 | I0030 전체 (신규 제품 포착) |
| Daily Sync | 매일 03:00 | I0030 최근 24시간 업데이트분 |
| 원료 기준 갱신 | 분기 1회 | 금지/제한 원료 목록 변경 확인 |
| 기능성 심사 | 주 1회 | I0040 신규 심사 결과 |

## 6. 제약사항 및 주의점

1. **성분 함량 미제공**: 식약처 API는 성분 목록만 제공하고 정확한 함량(%)은 미제공. 전성분 표시 순서가 함량 순이므로 orderIndex로 대략 추정.
2. **성분명 비표준화**: 한국명/INCI명 혼용, 동일 성분의 다양한 표기. IngredientSynonym 테이블 적극 활용 필요.
3. **카테고리 매핑 한계**: 식약처 카테고리(기초화장품, 색조화장품 등)가 COSFIT ProductCategory와 1:1 매핑되지 않음. 세분화 로직 필요.
4. **수입 화장품 누락**: 일부 수입 화장품은 식약처 등록이 늦거나 누락될 수 있음. 보완 데이터 소스 필요.
5. **API 장애**: 식품안전나라 서버 점검(주로 일요일 새벽) 시 일시 중단. 재시도 로직 필수 (현재 구현 완료).
